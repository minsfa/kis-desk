# 지표 분석 프레임 — 값이 아니라 "의미"로 본다

종목을 *숫자*로 보지 않고 **지표 + 해석 기준(threshold)** 으로 본다. 도구는 raw 값이 아니라
"좋음/주의/나쁨" 같은 *의미*를 출력한다(가치 쪽 🪤/🟢처럼). 두 개의 렌즈를 함께 쓴다.

- 🟦 **가치 렌즈** = "싼가?" (자산·현금흐름 대비) — *이미 구현*
- 🟧 **성장·혁신 렌즈** = "더 커질 종목인가?" (ARK식) — *신규*

각 종목은 **둘 다** 받아, 성격에 맞는 렌즈로 판단한다(가치주는 🟦, 성장주는 🟧, 사이클주는 둘 다).

---

## 🟦 가치 렌즈 (구현됨)
| 지표 | 의미 | 해석 기준 | 출처 |
|---|---|---|---|
| PBR | 주가÷순자산 | 지주사엔 부적합(취득원가 착시)→NAV로 | KIS |
| PER | 주가÷이익 | 사이클 바닥이면 높아 보이는 착시 주의 | KIS |
| **NAV 할인율** | 시총÷보유 상장지분 시가 | +40%↑ 견고 / 음수+저cov=아티팩트 / 음수+고cov=프리미엄 | DART+KIS |
| ROE | 자기자본수익률 | ≥15 우수·10~15 양호·<5 약함·<0 적자 | KIS |
| 부채비율 vs **차입의존도** | 레버리지 | 금융·보험은 부채비율 무시, 차입의존도로 | KIS |
| 외국인 수급 | 30일 순매수 | 전환>순매수>매도, 대량매도=이탈경고 | KIS |
| 자사주 | 주주환원 강도 | **소각 > 공개매수 > 취득 > (처분=환원 아님)** | DART |
| 🪤 가치함정 | 싸다+본업부실 | 저PBR/고NAV할인 + (저ROE or 적자 or 감익 or 고차입) | 종합 |

## 🟧 성장 렌즈 — 우리가 수치화하는 것 (KIS로 계산)
정성 영역(TAM·해자·경영진·라이트의 법칙)은 **도구가 흉내내지 않는다** → 윤기 님이 별도 딥리서치.
도구는 **KIS로 신뢰성 있게 뽑히는 정량 지표 + 해석 라벨**만 출력한다.

| 지표 | 의미 | 해석 기준 | 비고 |
|---|---|---|---|
| **매출 CAGR**(가용 연수) | 외형 성장 속도 | ≥20% 고성장·10~20 견조·<10 저성장 | ✅ KIS 손익(연간) |
| 영업이익률 + 추세 | 본업 수익성 방향 | 확대=스케일 효과 / 축소=경쟁·비용 | ✅ KIS |
| 순이익률 | 최종 마진 | — | ✅ KIS |
| ROE | 자본효율 | ≥15 우수·<5 약함·<0 적자 | ✅ KIS |
| **PSR**(주가매출) | 적자·성장기 밸류 | 단독 X | ✅ KIS |
| **성장조정 PSR** = PSR ÷ 매출성장% | "매출 PEG" | <1 매력·1~2 보통·>3 부담 | ✅ 계산 |
| **PEG** = PER ÷ 이익성장% | 성장 대비 PER | <1 저평가·>2 부담 | ✅ 계산 |
| **5년 15% CAGR 허들** | ARK 매수 최소선 | 현재가→5년뒤 목표시총 연복리 <15%면 탈락 | ✅ 계산 |
| **멀티플 컴프레션** | 보수성 테스트 | 미래 배수를 낮춰 가정해도 15% 나오나 | ✅ 계산 |

## 별도(우리 수치화 안 함 — 윤기 님 딥리서치 영역)
- Gross Margin(KIS 부정확)·R&D 강도 → 필요 시 *DART 2단계*로 후순위 검토
- **TAM·침투율 / 라이트의 법칙 / 해자(Moat) / 경영진·문화 / 제품 리더십·센티먼트 / 테시스 리스크** → 정성, web·리포트로 직접.
- 매크로(생산성·Trueflation) → 레짐 필터(별도 트랙)
> 원칙: ❌ 정성 지표를 가짜 숫자로 만들지 않는다(거짓 정밀 금지).

## 적용 원칙
- **렌즈를 종목 성격에 맞춰라.** 가치주(SK·현대해상)=🟦 / 성장주(크래프톤·툴젠)=🟧 / 사이클(삼성SDS·물류)=둘 다(바닥이익 PER 착시 주의).
- **추세 > 절대값.** Gross margin·영업이익률·CAGR은 *방향*이 핵심.
- **사이클 바닥 PER/PSR 착시 경계.** 분모(이익·매출)가 바닥이면 배수가 부풀어 보임 → 정상화 이익으로 재계산.
- **❌ 지표는 흉내내지 않는다.** TAM·Wright's·ARR은 정성 참고로만, 가짜 숫자 만들지 않는다.
