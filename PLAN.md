"자동매매 봇"에서 **"텔레그램(OpenClaw/madu_bot)에서 묻고·답받고·필요시 매매하는 KIS 투자 코파일럿"** 으로 진화.
- 🔬 **리서치 엔진** = *무엇을 살까*를 데이터로 답 (지주사 NAV/스코어카드/펀더멘털/공시/수급). → 대화로 자유롭게.
- ⚙️ **실행 엔진** = *사라*를 안전하게 수행 (승인 게이트·한도·킬스위치). → **감독형 오토파일럿**: 사람이 승인해야 작동.
- 💬 **창구** = OpenClaw 플레이북(`~/.openclaw/workspace/KIS_*.md`)이 자연어 요청을 명령으로 매핑.

비유: 리서치 = 언제든 답하는 **내비게이션**, 매매 = 사람이 손 올리고 승인해야 도는 **오토파일럿(테슬라식)**.

## 1. 핵심 방침 (2026-06-03 확정)
**전략 방향(레버리지 단타 vs 거시 스윙 vs 도미노 오버레이)은 아직 정하지 않는다.**
대신 **무엇을 하든 필요한 공통 인프라 — KIS API 기반(모의·실전 양쪽 동작) — 을 먼저 만들어 놓고**, 그 위에서 전략을 판단한다.

→ 즉 이번 목표(M1)는 "전략"이 아니라 "**토큰·시세·잔고·주문이 모의/실전 양쪽에서 안전하게 도는 토대**"다.

## 2. 지금까지의 핵심 발견 (의사결정 근거)
- **레버리지 ETF 일중 단타**: 개인 누적손실 등 음(-)의 기대값 증거 강함. 손익분기 승률 55% 필요. KIS는 과거 분봉 미제공 + 모의투자가 단타 검증에 부적합 → **검증 자체가 어려움**. → **보류 트랙**(분봉 DB·백테스트 인프라 갖춰진 뒤 실험).
- **거시/도미노 신호**: 잘 설계된 모델이나, KR 진입신호가 아니라 **리스크 레짐 필터**. 게다가 현재 이력 ~1일 + 파이프라인 끊김(엔진 `~/domino`가 heartbeat에서 분리, 텔레그램 리포트는 LLM이 손으로 작성) → **별도 수리 필요**.
- **결론**: 전략은 미정으로 두고 **API 토대부터**. 토대가 서면 전략은 데이터로 비교해 고른다.

## 4. M1 상세 설계 — KIS API 기반 (모의·실전 양쪽)

### 코드 구조
```
kis-autotrade/
├── src/
│   ├── kis/
│   │   ├── config.py    환경(vts/prod) 스위치, 경로, .env 로드
│   │   ├── auth.py      접근토큰 발급·파일캐시·재발급 스로틀
│   │   ├── client.py    REST 공통 클라이언트 + rate limit + 헤더
│   │   ├── market.py    현재가·잔고 조회
│   │   ├── orders.py    매수/매도 (hashkey + 안전게이트)
│   │   ├── safety.py    한도·킬스위치·일일카운터
│   │   └── logging_util.py  전 호출/주문 JSONL 로깅
│   └── cli.py           진입점: price/balance/buy/sell, --env, --dry-run/--live
├── config/.env          키 (git 제외)
└── logs/                JSONL 감사로그
```

### 환경 스위치 (핵심 설계)
- `KIS_ENV=vts`(모의) / `prod`(실전). CLI `--env`로 override. **기본 vts.**
- 도메인: 모의 `https://openapivts.koreainvestment.com:29443` / 실전 `https://openapi.koreainvestment.com:9443`.
- TR ID 접두: 모의 `V***` / 실전 `T***` 를 환경에 따라 자동 선택.
- 키·계좌·토큰캐시를 환경별로 완전 분리(모의 키로 실전 주문 불가).

### 안전장치 (M1부터 내장)
1. **DRY_RUN 기본 true** — 실주문 대신 "보낼 내용"만 출력/로깅. `--live`로만 실주문.
2. **하드 한도** — 1회 주문 최대금액(`MAX_ORDER_AMOUNT`), 일일 최대 매매횟수(`MAX_DAILY_TRADES`). 초과 시 차단.
3. **킬스위치** — 프로젝트 루트에 `STOP` 파일 존재 시 모든 주문 차단.
4. **일일 카운터** — `state/daily_count_<date>.json`로 매매 횟수 추적.
5. **전수 로깅** — 모든 토큰/시세/주문/에러를 `logs/YYYY-MM-DD.jsonl`에 기록.

### ⚠️ 빌드 직전 공식 확인 항목 (코드에 TODO 표기)
- 현금주문 매수/매도 **TR ID** (조사상 매수 `TTTC0802U`/`VTTC0802U`, 매도 `TTTC0801U`/`VTTC0801U` — **공식 포털 재확인**).
- 잔고 `TTTC8434R`/`VTTC8434R`, 호출 한도(모의 2/s·실전 20/s), 분봉 제한.
- 국내 레버리지 ETF **기본예탁금**(증권사 의존, 한투 직접 확인) 및 ETP 사전교육 — 실전 주문 전 필수.
